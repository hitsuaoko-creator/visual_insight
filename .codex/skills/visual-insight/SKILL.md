# 视觉洞察

## 使用方式

当用户输入 `/visual-insight`、`使用 visual-insight skill`，或提出视觉洞察相关需求时，按本 Skill 执行。

## 必读知识库

本 Skill 的通用知识源在仓库根目录或安装目录：

- `docs/视觉洞察工作流.md`
- `docs/视觉原则知识库.md`
- `docs/竞品UXUI分析框架.md`
- `docs/输出模板.md`
- `prompts/视觉洞察主Prompt.md`

如果当前目录下没有 `docs/`，按以下顺序查找：

1. 当前 workspace 根目录的 `docs/`
2. 当前 Skill 目录向上三级后的 `docs/`，即项目内适配路径 `../../../docs/`
3. 当前 Skill 安装目录的 `docs/`

如果仍然找不到，提示用户在仓库根目录运行 `bash scripts/install.sh` 重新安装完整包。

## 强制执行

1. 先阅读 `docs/视觉洞察工作流.md` 和 `docs/视觉原则知识库.md`。
2. 如果包含竞品分析，阅读 `docs/竞品UXUI分析框架.md`。
3. 如果需要结构化交付，阅读 `docs/输出模板.md`。
4. 建立“业务目标 - 路径步骤 - UI 元素 - 设计原则 - 证据 - 指标”的映射。
5. 每条重要建议尽量包含问题/机会点、证据来源、路径步骤、相关 UI 元素、UI 设计动作、命中原则、预期影响、验证指标和优先级。

## 约束

- 不编造竞品截图中不存在的细节。
- 不把设计原则当百科解释。
- 不输出“提升高级感”“优化体验”“更清晰”等空话，除非转成具体 UI 动作。
- 材料不足时不要停止，先标注判断边界，再输出当前可判断内容。
