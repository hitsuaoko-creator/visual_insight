# CLAUDE.md

## 使用方式

当用户要求视觉洞察、UI 视觉分析、竞品 UXUI 分析、基于 PRD 的视觉设计洞察、设计前置 Checklist、UI 方案前分析、视觉走查或设计评审前分析时，按本文件执行。

## 必须读取

先读取：

1. `docs/视觉洞察工作流.md`
2. `docs/视觉原则知识库.md`

如果任务包含竞品分析，再读取：

3. `docs/竞品UXUI分析框架.md`

如果用户需要完整结构化输出，再读取：

4. `docs/输出模板.md`

如需给用户一个可复用 Prompt，读取：

5. `prompts/视觉洞察主Prompt.md`

如果涉及输入法 APP、MasterGo、组件库、视觉设计 Skill、设计规范、C2D，或洞察报告会作为视觉审查输入，读取：

6. `docs/设计落地约束洞察.md`

## Claude Code 执行约束

- 将 `docs/` 视为唯一知识源。
- 不要复制维护另一份原则库。
- 不要输出泛泛审美建议。
- 每条关键建议都要尽量引用知识库中的原则。
- 如果某条建议来自业务判断，而非原则库直接推导，明确标注。
- 如果某条建议来自设计规范、组件库或视觉设计 Skill，标注“设计落地约束，不属于原则库直接推导”。
- 如果洞察报告会作为视觉审查输入，高优先级建议尽量写出后续视觉审查检查点。
- 若用户材料不足，先说明已提供、缺失、可判断和需补充内容，再继续分析。

## 输出

优先按 `docs/输出模板.md` 输出。若用户只要求简短结论，可压缩模板，但保留：

- 需求视觉目标
- 关键视觉路径
- 命中原则
- 可执行设计建议
