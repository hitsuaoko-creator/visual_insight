## 输出要求

- “设计原则命中分析”必须排在第四章。
- “关键视觉路径”排在第五章。
- “竞品 UXUI 洞察”只用一张主表承载竞品相关性、可借鉴点与风险点。
- 竞品表中“可借鉴点”和“风险点”放在第二、第三列。
- 竞品章节不要拆出额外的竞品判断小标题或多张主表。
- “当前 UI / 链路视觉洞察”和“视觉洞察 Checklist”必须合并为一张 Checklist 表。
- Checklist 表必须保留：维度、当前判断、建议动作、当前需求需要重点关注什么、证据来源、优先级。
- 每条建议必须能指导 UI 设计动作。
- 每条建议尽量绑定原则和指标。
- 如果属于业务判断，标注“业务判断，不属于原则库直接推导”。
- 如果属于设计规范、组件库或视觉设计 Skill 规则推导，标注“设计落地约束，不属于原则库直接推导”。
- 如果视觉洞察报告会作为视觉审查输入，每条高优先级建议必须尽量写出“后续视觉审查检查点”。
- 优先级使用：高 / 中 / 低。
