# 竞品 UXUI 分析框架

## 目标

竞品分析必须服务当前需求，不做泛泛的产品盘点。每条洞察都要能回到当前业务目标、关键路径、UI 元素和设计指标。

## 输入材料

优先需要：

- 当前需求 PRD
- 关键路径
- 目标角色
- 竞品名称
- 竞品截图或链接
- 当前产品截图或历史版本
- 品牌规范或历史 UI 问题

如果没有竞品截图，不要编造。输出竞品采集清单。

## 竞品 UXUI 洞察表

分析竞品时，重点看：

- 入口如何被看见
- 首屏如何建立价值
- 信息层级如何组织
- 主 CTA 如何呈现
- 卡片、列表、表单如何降低理解成本
- 关键状态如何反馈
- 风险、价格、权限、隐私如何建立信任
- 成功、失败、空状态如何闭环
- 品牌调性是否稳定
- 是否存在同质化或不适合本产品的问题

最终报告只输出一张竞品 UXUI 洞察主表。不要拆成多个竞品判断小标题或多张主表。

表格字段固定为：

| 竞品名称 | 可借鉴点 | 风险点 | 与本需求的相关场景 | 类型 | 适合分析的页面 / 路径 | 为什么相关 |
|---|---|---|---|---|---|---|
| ... | ... | ... | ... | 直接竞品 / 间接竞品 / 标杆竞品 / 反例竞品 | ... | ... |

字段填写要求：

- **竞品名称**：写具体产品或页面来源，不写泛称。
- **可借鉴点**：必须能转成当前需求的 UI 动作，例如首焦组织、卡片层级、数据表达、CTA 位置、状态反馈、信任信息呈现。
- **风险点**：说明该做法迁移到当前产品可能造成的问题，例如品牌割裂、信息过载、误导转化、弱化风险提示、操作成本增加。
- **与本需求的相关场景**：写当前需求中的对应场景或路径步骤，不写“同类产品”这种泛泛理由。
- **类型**：只用直接竞品、间接竞品、标杆竞品、反例竞品。
- **适合分析的页面 / 路径**：写具体页面或链路，例如数据首页、作品详情、收益明细、空状态、发布成功页。
- **为什么相关**：说明它和当前业务目标、用户判断或 UI 问题的关系。

如需在分析过程中拆路径，可以在草稿中使用以下观察维度，但最终报告仍合并到上面的主表：

| 观察维度 | 重点判断 |
|---|---|
| 入口 | 入口是否可见、是否能说明价值、是否干扰主任务 |
| 首焦 | 第一眼是否落在当前需求最关键的信息或行动上 |
| 信息层级 | 重点数据、任务、风险、说明是否有稳定主次 |
| 主 CTA | 位置、文案、视觉权重是否支持下一步行动 |
| 视觉动线 | 用户是否能自然从理解价值走到操作闭环 |
| 状态反馈 | 加载、成功、失败、空状态是否完整 |
| 品牌调性 | 是否符合当前产品的品牌心智与设计系统 |
| 效率设计 | 是否减少步骤、减少判断成本、降低误触 |
| 风险点 | 是否有不适合当前业务、品牌或用户心智的表达 |

## 可借鉴视觉策略

每条按以下格式输出：

```text
- 洞察：
- 来源竞品/截图证据：
- 对应当前需求路径：
- 可转化为本产品的 UI 动作：
- 命中设计原则：
- 对应指标：
- 注意事项：
```

## 应规避内容

明确说明哪些竞品做法不适合当前品牌、当前用户或当前业务目标。

常见规避点：

- 首屏只有氛围，没有任务入口。
- 所有信息平铺，缺少主次。
- 为了视觉冲击牺牲可读性。
- 主 CTA 和次要按钮竞争。
- 高风险信息被弱化。
- 状态反馈不完整。
- 风格和当前产品品牌心智割裂。

## 没有竞品材料时的输出

```text
当前未提供竞品截图或链接，因此不做具体竞品结论。

建议采集以下竞品页面：
- 入口页：看功能如何被看见
- 首屏页：看如何建立价值判断
- 关键操作页：看主 CTA、表单、卡片、列表的表达
- 状态页：看加载、成功、失败、空状态
- 结果页：看如何承接下一步

采集重点：
- 首焦
- 信息层级
- 视觉动线
- 主 CTA
- 状态反馈
- 风险与信任
- 品牌一致性
```
