# 视觉洞察工作流

## 目标

把 PRD、需求背景、用户角色、交互链路截图、当前 UI、竞品 UXUI 材料、品牌规范和历史 UI 问题，转成可进入 UI 方案设计与设计评审的视觉洞察。

本工作流不负责直接画 UI，也不输出泛泛审美建议。它要求每条关键判断都尽量绑定：

- 业务目标
- 用户角色与场景
- 关键路径步骤
- 具体 UI 元素
- 视觉设计原则
- 证据来源
- 预期影响
- 验证指标

## 输入材料

优先收集：

- 需求名称
- PRD
- 需求背景
- 目标角色
- 关键路径
- 交互链路截图
- 当前 UI 或历史版本截图
- 竞品名称、截图或链接
- 品牌规范、设计系统或品牌关键词
- 历史 UI 问题
- 业务指标或评审关注点

## 输入模式

### 完整输入模式

用户已提供 PRD、背景、角色、关键路径、链路截图、竞品材料、品牌规范和历史 UI 问题。

处理方式：直接输出完整视觉洞察报告。

### 半完整输入模式

用户只提供 PRD、背景、关键路径或部分截图。

处理方式：

- 先说明已提供与缺失材料。
- 基于已有信息先做判断。
- 明确哪些判断需要补充材料后才能完成。
- 如果缺少竞品材料，输出竞品采集清单。

### 探索输入模式

用户只提供一句话需求或模糊方向。

处理方式：

- 不直接生成完整结论。
- 先反推可能的业务目标、用户角色和关键路径。
- 输出需要补充的材料清单。
- 输出初步视觉洞察框架。

## 引导式提问

如果材料不足，一次最多问 4 个问题，优先问影响判断最大的内容：

1. 这个需求的 PRD 或需求背景是什么？
2. 关键路径是什么？
3. 有没有链路截图、旧版 UI 或交互稿？
4. 是否需要包含竞品 UXUI 分析？

如果用户不想补充，继续基于已有材料输出，并标注判断边界。

## 材料不足时的输出格式

```text
已提供材料：
- ...

缺失材料：
- ...

基于现有材料可以判断：
- ...

补充后可以进一步判断：
- ...

建议下一步补充：
- ...
```

## 强制知识库巡视流程

进行视觉洞察前，必须先巡视知识库，而不是直接凭审美给结论。

### 第一步：读取通用知识库

必须读取：

1. `docs/视觉洞察工作流.md`
2. `docs/视觉原则知识库.md`

如果任务包含竞品分析，读取：

3. `docs/竞品UXUI分析框架.md`

如果用户需要结构化交付物，读取：

4. `docs/输出模板.md`

如果需要可复制 Prompt，读取：

5. `prompts/视觉洞察主Prompt.md`

### 第二步：建立“原则 - 证据”映射

每一次视觉洞察任务，都必须先判断当前需求命中了哪些视觉设计原则，并建立以下映射：

- 业务目标
- 用户角色与使用场景
- 关键路径步骤
- 涉及的 UI 元素
- 命中的设计原则
- 证据来源：PRD / 链路截图 / 当前 UI / 竞品截图 / 历史 UI 问题
- 对用户判断或行为的影响
- 可关联的设计指标

只有当某条设计原则能够对应到具体业务路径、具体 UI 元素或具体用户判断时，才允许使用这条原则。不要为了显得专业而硬套原则。

### 第三步：用原则作为判断标准

重点巡视：

- 第一眼落点是否正确
- 信息层级是否服务业务优先级
- 视觉动线是否能引导用户进入下一步
- 主 CTA 是否清晰、稳定、易识别
- 页面分组和布局是否降低理解成本
- 高频操作是否便捷、易触达
- 状态反馈是否完整
- 加载、成功、失败、空状态是否形成闭环
- 是否提前预防错误和误触
- 是否符合品牌一致性
- 是否能建立信任感和安全感
- 文案、字号、对比度是否具备可读性
- 移动端是否适配单手操作和小屏场景
- 是否修复历史 UI 问题

不要把原则当成理论解释来输出。必须把原则转化为具体 UI 观察、具体问题判断或具体设计动作。

### 第四步：输出有依据的视觉洞察

每一条重要视觉建议都必须包含：

- 问题 / 机会点
- 证据来源
- 对应路径步骤
- 相关 UI 元素
- 建议的 UI 设计动作
- 命中的设计原则
- 预期影响
- 关联设计指标
- 优先级

如果某条建议主要来自业务判断，而不是原则库直接推导，标注：

> 该建议属于业务判断，不属于原则库直接推导。

## 原则索引

### 首屏、入口、弹窗、任务开始页

优先检查：

- 首焦原则
- 视觉层级原则
- 对比原则
- 古腾堡原则
- Z 型动线
- 主 CTA 识别

### 列表、搜索结果、内容管理、商品列表、B 端表格

优先检查：

- F 型浏览
- 视觉层级原则
- 接近原则
- 相似原则
- 分块原则
- 识别优于记忆

### 表单、设置页、配置页、提交页

优先检查：

- 接近原则
- 连续原则
- 系统状态可见
- 错误预防原则
- 识别优于记忆
- Fitts 定律

### 支付、授权、隐私、交易、会员、发布

优先检查：

- 信任强化原则
- 错误预防原则
- 系统状态可见
- 视觉层级原则
- 图底关系原则
- 一致性原则

### AI 生成、上传、保存、审核、长任务

优先检查：

- 系统状态可见
- 连续原则
- Peak-End 体验
- 错误预防原则
- 操作闭环
- 状态反馈完整性

### 首页、活动页、会员权益页、轻量转化页

优先检查：

- Z 型动线
- 古腾堡原则
- 首焦原则
- 视觉层级原则
- 对比原则
- 分块原则

### 移动端高频操作、单手场景、碎片时间任务

优先检查：

- Fitts 定律
- 单手操作原则
- 响应式适配原则
- 主 CTA 识别
- 误触预防

### 品牌刷新、新功能接入、跨页面一致性

优先检查：

- 品牌一致性原则
- 一致性原则
- 色彩可用性原则
- 可读性原则
- 历史 UI 问题修复

## 指标到原则映射

- 主 CTA 识别率：首焦原则、对比原则、视觉层级、Fitts 定律
- 页面转化率：视觉动线、主 CTA 识别、信任强化、Hick 定律
- 任务完成率：连续原则、系统状态可见、错误预防、识别优于记忆
- 操作步数：Hick 定律、分块原则、识别优于记忆
- 完成时长：F 型浏览、视觉层级、分块原则、Hick 定律
- 路径流失率：视觉动线、状态反馈、信任强化、操作闭环
- 错误率：错误预防、可读性、色彩可用性、识别优于记忆
- 误触率：Fitts 定律、相似原则、危险操作隔离
- 首屏吸引力：首焦原则、视觉层级、Z 型动线
- 卡片点击率：共同区域、视觉层级、对比原则
- Banner 点击率：首焦原则、Z 型动线、对比原则
- 品牌一致性：品牌一致性原则、一致性原则、色彩可用性
- 视觉喜好度：可读性、图底关系、信息降噪、品牌一致性

## 禁止泛化表达

禁止只写：

- 提升高级感
- 优化体验
- 强化氛围
- 增强视觉吸引力
- 更加简洁
- 更有品质感
- 更年轻化
- 更专业
- 更清晰
- 更好看

除非同时说明：具体改哪个 UI 元素、为什么改、服务哪个路径步骤、解决什么用户判断问题、对应哪个设计原则、影响哪个指标。
