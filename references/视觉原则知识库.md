# 视觉原则知识库

## 结构

每条原则按以下方式使用：

- 适用场景
- UI 重点
- 洞察时要看什么
- 常见问题
- 可转化的设计动作
- 关联指标

不要把本文件当百科解释。输出时只引用和当前需求有关的原则，并转成具体 UI 判断。

## 一、视觉注意力原则

### 1. 首焦原则

适用场景： 首页、详情页、弹窗、功能入口页、支付页、发布页、任务开始页。

UI 重点：

- 用户第一眼应该看到什么。
- 首焦是否服务当前需求的核心目标。
- 主信息、关键状态、主 CTA 是否在首屏强感知区域。

洞察时要看：

- 第一眼落点是主任务，还是被插画、Banner、弱相关运营位抢走。
- 页面最强视觉元素是否等于最重要业务元素。
- 首屏是否能在 3 秒内回答：这是什么、对我有什么用、下一步点哪里。

常见问题：

- 装饰图比主 CTA 更抢眼。
- 标题很大，但没有表达用户收益。
- 主按钮被放在弱视觉区。
- 首屏信息太平均，用户不知道先看哪里。

设计动作：

- 强化主 CTA 对比度、尺寸、位置。
- 降低次要元素饱和度、面积和位置权重。
- 用标题、关键利益点、主按钮形成连续阅读路径。

关联指标： 主 CTA 识别率、页面转化率、首屏吸引力、路径流失率。

### 2. 视觉层级原则

适用场景： 所有复杂页面，尤其是卡片、表单、详情页、列表、运营页、后台页面。

UI 重点：

- 信息必须有主次。
- 视觉层级要与业务优先级一致。

洞察时要看：

- 一级信息：用户必须马上知道的内容。
- 二级信息：辅助判断的信息。
- 三级信息：补充说明、规则、备注。
- CTA、状态、价格、权益、风险提示是否有正确权重。

常见问题：

- 字号、颜色、字重全部接近。
- 标签、说明、按钮抢同一层级。
- 重要信息被灰色小字弱化。
- 视觉上像“排版完成”，但用户无法判断优先级。

设计动作：

- 建立标题、核心信息、辅助信息、操作区的层级比例。
- 用字号、字重、色彩、留白、容器共同拉开层级。
- 对弱信息做收纳、折叠或降噪。

关联指标： 任务完成率、完成时长、错误率、页面转化率。

### 3. 对比原则

适用场景： 按钮、状态、价格、权益、风险提示、选中态、错误态。

UI 重点：

- 通过明暗、大小、色彩、形状、空间制造差异。
- 对比必须服务判断，不只是为了“更醒目”。

洞察时要看：

- 主按钮和次按钮是否一眼可区分。
- 选中态和未选中态是否明显。
- 错误、警告、成功、禁用状态是否有稳定视觉语言。
- 关键数字、价格、权益是否被有效突出。

常见问题：

- 主次按钮都很亮。
- 风险提示和普通说明长得一样。
- 选中态只靠细小边框区分。
- 灰色层级过多，信息变脏。

设计动作：

- 主动作使用最高视觉权重。
- 次动作降为文本按钮或弱描边。
- 风险信息使用颜色、图标和文案共同表达。
- 状态色固定，不混用。

关联指标： 主 CTA 识别率、误触率、错误率、任务完成率。

## 二、视觉动线原则

### 4. 古腾堡原则

适用场景： 低信息密度页面、落地页、弹窗、确认页、单一任务页。

UI 重点：

- 用户视线通常从左上进入，向右下收束。
- 右下区域适合放置最终动作。

洞察时要看：

- 左上是否承载核心标题或任务起点。
- 右下是否有主 CTA 或下一步操作。
- 中间区域是否支撑用户理解，而不是塞满无关信息。

常见问题：

- 重要入口放在左下或边缘区域。
- 主 CTA 不在动线终点。
- 用户看完内容后找不到下一步。

设计动作：

- 让标题、核心信息、操作按钮形成左上到右下的路径。
- 弹窗中把主按钮放在用户自然扫视后的末端。
- 减少动线中的干扰点。

关联指标： 页面转化率、主 CTA 识别率、完成时长。

### 5. Z 型动线

适用场景： 首页首屏、活动页、会员权益页、产品介绍页、轻量转化页。

UI 重点：

- 适合信息量不大、需要快速建立认知并引导点击的页面。
- 常见路径：品牌或标题 → 核心利益 → 视觉内容 → CTA。

洞察时要看：

- 顶部是否承担识别和核心价值。
- 中段是否用图像、卡片或权益强化判断。
- 末端是否明确收束到主操作。

常见问题：

- 页面视觉很丰富，但没有动线收束。
- CTA 出现过早或过晚。
- 用户看到利益点后没有对应操作。

设计动作：

- 用标题、权益、图像、CTA 排成 Z 型节奏。
- 控制每一段只承担一个判断任务。
- CTA 在动线末端重复强化。

关联指标： 入口点击率、页面转化率、首屏吸引力。

### 6. F 型浏览

适用场景： 列表页、搜索结果页、内容管理页、商品列表、B 端表格。

UI 重点：

- 用户会优先扫左侧标题、首行信息、开头字段。
- 适合高信息密度和快速比较。

洞察时要看：

- 每行最重要的信息是否靠左、靠前。
- 列表是否支持快速扫描和比较。
- 标签、状态、价格、时间是否位置稳定。

常见问题：

- 重要状态放在卡片右下角。
- 列表每项结构不一致。
- 用户需要来回扫视才能比较。

设计动作：

- 把名称、状态、核心指标放在左侧或首行。
- 保持卡片、列表字段位置一致。
- 次级操作收纳到右侧或更多菜单。

关联指标： 完成时长、操作步数、任务完成率、错误率。

## 三、格式塔原则

### 7. 接近原则

适用场景： 表单、卡片、设置页、筛选项、权益说明、订单确认页。

UI 重点：

- 距离近的元素会被用户理解为一组。
- 分组关系要靠空间表达，而不是只靠线框。

洞察时要看：

- 标题、说明、输入框、错误提示是否离得足够近。
- 不同模块之间是否有足够间距。
- 用户是否能快速判断哪些内容属于同一个任务。

常见问题：

- 说明文案离对应控件太远。
- 两个模块之间间距太近，造成误解。
- 表单错误提示位置不稳定。

设计动作：

- 组内小间距，组间大间距。
- 错误提示紧跟对应输入项。
- 卡片内信息按语义分区。

关联指标： 错误率、完成时长、任务完成率。

### 8. 相似原则

适用场景： 按钮、状态标签、列表、功能入口、卡片、导航。

UI 重点：

- 长得一样的元素会被认为功能相同。
- 不同功能的元素不能使用过于相似的视觉样式。

洞察时要看：

- 主按钮、次按钮、危险按钮是否样式区分明确。
- 标签、徽章、按钮是否混用同一种样式。
- 可点击和不可点击元素是否容易混淆。

常见问题：

- 标签像按钮。
- 普通文字像链接。
- 危险操作和普通操作样式接近。

设计动作：

- 建立按钮、标签、状态、提示的独立样式系统。
- 危险操作使用独立颜色和确认机制。
- 可点击元素保持一致反馈。

关联指标： 误触率、错误率、任务完成率。

### 9. 共同区域原则

适用场景： 卡片、表单分区、后台模块、配置项、订单信息。

UI 重点：

- 被同一个容器包住的内容会被认为属于同一组。
- 容器不是越多越好，必须服务理解。

洞察时要看：

- 卡片边界是否帮助用户理解模块。
- 是否存在卡片套卡片，导致层级混乱。
- 容器是否掩盖了真正的信息主次。

常见问题：

- 所有内容都装进卡片，页面碎片化。
- 容器边框太重，抢走内容注意力。
- 分区太多，用户无法判断重点。

设计动作：

- 用轻背景、留白、分割线建立区域。
- 只给真正需要独立判断的内容加容器。
- 同一业务对象保持一个主要容器。

关联指标： 完成时长、任务完成率、视觉喜好度。

### 10. 连续原则

适用场景： 流程页、步骤条、表单、引导页、支付链路、发布链路。

UI 重点：

- 用户倾向沿着连续路径理解信息。
- 视觉排列要帮助用户知道当前在哪、下一步去哪。

洞察时要看：

- 步骤之间是否有连续关系。
- 当前步骤、已完成步骤、下一步骤是否清楚。
- 页面滚动后操作区是否还能承接。

常见问题：

- 步骤条弱，用户不知道进度。
- 表单段落断裂，像多个无关模块。
- 底部 CTA 与内容关系弱。

设计动作：

- 使用步骤条、编号、连续线、固定操作栏。
- 每段表单后给出明确下一步。
- 长流程中强化当前位置和完成反馈。

关联指标： 路径流失率、任务完成率、完成时长。

### 11. 图底关系原则

适用场景： 弹窗、浮层、Toast、引导蒙层、复杂背景上的文字。

UI 重点：

- 用户必须清楚哪个是内容主体，哪个是背景。
- 前景与背景不能竞争。

洞察时要看：

- 弹窗是否足够聚焦。
- 背景是否被有效弱化。
- 图片上的文字是否可读。
- 悬浮按钮是否和底层内容混在一起。

常见问题：

- 透明背景导致文字难读。
- 弹窗和页面背景层级不明显。
- 浮层阴影过弱，没有空间关系。

设计动作：

- 增加遮罩、模糊、阴影或背景降噪。
- 图片上加渐变遮罩或固定文字安全区。
- 浮层使用明确层级样式。

关联指标： 错误率、任务完成率、视觉喜好度。

## 四、认知效率原则

### 12. Hick 定律

适用场景： 首页入口、工具栏、筛选器、弹窗、设置页、创建流程。

UI 重点：

- 选择越多，用户决策越慢。
- UI 需要控制同时暴露的选项数量。

洞察时要看：

- 当前步骤是否给了过多选择。
- 主任务是否被多个平级按钮稀释。
- 是否需要分组、默认推荐、渐进展开。

常见问题：

- 一个页面出现多个主按钮。
- 入口太多，用户不知道从哪开始。
- 高级选项过早暴露。

设计动作：

- 保留一个主 CTA。
- 次级功能收进更多、折叠或二级页。
- 给复杂选项增加推荐项、默认项、分组标题。

关联指标： 完成时长、页面转化率、操作步数。

### 13. Fitts 定律

适用场景： 移动端按钮、工具栏、底部操作区、高频功能、滑动控件。

UI 重点：

- 越大、越近、越稳定的目标越容易点击。
- 高频操作应放在舒适操作区。

洞察时要看：

- 主按钮是否足够大。
- 手机端是否方便单手点击。
- 危险操作是否离主操作过近。
- 点击区域是否大于视觉区域。

常见问题：

- 图标按钮太小。
- 两个相反操作距离太近。
- 底部按钮被系统手势区干扰。

设计动作：

- 增大点击热区。
- 主操作固定在底部或任务末端。
- 危险操作远离高频操作。

关联指标： 误触率、任务完成率、完成时长。

### 14. 分块原则

适用场景： 权益页、套餐页、表单、设置页、数据看板、规则说明。

UI 重点：

- 用户无法一次处理太多信息。
- 复杂内容要按任务、语义或优先级分块。

洞察时要看：

- 信息是否按用户决策顺序组织。
- 每个模块是否只解决一个问题。
- 长文本是否被拆成可扫描结构。

常见问题：

- 规则说明堆成整段文字。
- 权益、价格、限制条件混在一起。
- 数据看板没有主题分区。

设计动作：

- 使用模块标题、短句、列表、分隔区。
- 把核心结论前置，细节后置。
- 一屏控制 3-5 个核心判断点。

关联指标： 完成时长、任务完成率、页面转化率。

### 15. 识别优于记忆

适用场景： 导航、表单、设置、复杂工具、编辑器、后台系统。

UI 重点：

- 用户不应该依赖记忆完成任务。
- UI 要把可选项、当前状态、历史选择展示出来。

洞察时要看：

- 用户是否需要记住上一步信息。
- 当前选择、默认值、历史记录是否可见。
- 图标是否有文字或 tooltip 辅助。

常见问题：

- 只给图标，不给含义。
- 表单提交前看不到已选内容。
- 多步骤流程中无法回看前面选择。

设计动作：

- 显示已选项、摘要、状态标签。
- 给低频图标加文字或提示。
- 关键配置提供预览。

关联指标： 错误率、完成时长、任务完成率。

## 五、可用性与反馈原则

### 16. 系统状态可见

适用场景： 上传、生成、支付、保存、发布、提交、加载、审核。

UI 重点：

- 用户需要知道系统正在做什么、做到哪一步、结果如何。

洞察时要看：

- 加载是否有明确反馈。
- 成功、失败是否清楚。
- 长任务是否有进度、预计时间或阶段反馈。
- 按钮点击后是否有状态变化。

常见问题：

- 点击按钮后无反馈。
- 加载态和禁用态长得一样。
- 成功后没有下一步承接。

设计动作：

- 增加 loading、进度条、步骤状态、Toast、结果页。
- 提交后按钮变为处理中。
- 成功态提供继续操作或查看结果。

关联指标： 错误率、路径流失率、二次使用率。

### 17. 错误预防原则

适用场景： 支付、删除、提交、权限授权、表单填写、批量操作。

UI 重点：

- UI 要提前降低用户犯错概率。
- 高风险操作要有更强视觉区分。

洞察时要看：

- 危险操作是否被弱化或隔离。
- 表单限制是否提前提示。
- 不可用按钮是否说明原因。
- 是否有二次确认和撤销机会。

常见问题：

- 删除按钮和普通按钮样式接近。
- 输入错误后才告诉用户规则。
- 禁用按钮没有解释。

设计动作：

- 输入前展示格式要求。
- 危险按钮使用独立样式。
- 提供实时校验、确认弹窗、撤销入口。

关联指标： 错误率、误触率、任务完成率。

### 18. 一致性原则

适用场景： 全局导航、组件库、按钮、状态、图标、列表、表单。

UI 重点：

- 相同功能使用相同样式。
- 相同状态使用相同视觉语言。
- 新需求不能破坏已有品牌和组件心智。

洞察时要看：

- 当前页面是否和产品其他页面风格割裂。
- 同一类按钮、标签、图标是否一致。
- 文案、圆角、阴影、间距是否符合设计系统。

常见问题：

- 为单个需求新造一套样式。
- 同一状态在不同页面颜色不同。
- 图标风格混杂。

设计动作：

- 复用现有设计 token 和组件。
- 新增样式前判断是否可沉淀为规范。
- 对核心状态建立统一颜色和图标。

关联指标： 品牌一致性、错误率、视觉喜好度。

## 六、品牌与信任原则

### 19. 品牌一致性原则

适用场景： 首页、活动页、会员页、AI 功能、支付页、核心业务路径。

UI 重点：

- 视觉风格必须服务品牌认知。
- 不只是颜色一致，还包括语气、图形、动效、质感、信息表达方式一致。

洞察时要看：

- 当前需求需要强化哪种品牌感：专业、可信、年轻、温暖、效率、创造力、安全。
- 是否延续品牌色、字体、图标、插画、圆角、阴影、动效。
- 是否因为追趋势导致品牌割裂。

常见问题：

- 活动页和主产品像两个产品。
- 新功能套用通用视觉表达，但没有品牌识别。
- 品牌色只用于装饰，没有用于关键路径。

设计动作：

- 明确本需求的品牌关键词。
- 将品牌色用于关键识别点，而不是全屏铺色。
- 建立符合品牌的图标、插画、动效语言。

关联指标： 品牌一致性、视觉喜好度、二次使用率。

### 20. 信任强化原则

适用场景： 支付、授权、隐私、交易、会员、金融、医疗、内容发布。

UI 重点：

- 高风险场景中，视觉要降低不确定感。
- 用户要清楚知道代价、收益、风险和结果。

洞察时要看：

- 价格、权限、隐私、协议、风险提示是否足够清楚。
- 关键信息是否被隐藏在弱层级。
- 是否有背书、说明、保障、可撤销机制。

常见问题：

- 主按钮很强，但风险说明很弱。
- 授权页没有解释为什么需要权限。
- 支付页信息层级混乱，用户缺乏确定感。

设计动作：

- 强化金额、权益、有效期、权限用途。
- 用图标、标签、说明建立安全感。
- 在关键动作前提供确认摘要。

关联指标： 页面转化率、错误率、路径流失率、品牌一致性。

## 七、可读性与无障碍原则

### 21. 可读性原则

适用场景： 所有包含文本的 UI，尤其是规则说明、表单、列表、弹窗、移动端页面。

UI 重点：

- 用户能不能轻松读完、扫完、理解。
- 字号、行高、对比度、行宽、信息密度都要服务阅读。

洞察时要看：

- 正文字号是否过小。
- 灰色文字是否对比不足。
- 长段文字是否缺少分段。
- 图片背景上的文字是否清晰。

常见问题：

- 重要说明用过小字号和浅灰色。
- 行距太密，阅读压迫。
- 一行太长，扫读困难。

设计动作：

- 正文保证足够字号和行高。
- 长文本拆成短句、列表或模块。
- 弱信息也要满足基础对比度。

关联指标： 完成时长、错误率、任务完成率。

### 22. 色彩可用性原则

适用场景： 状态、图表、标签、按钮、警告、选中态。

UI 重点：

- 色彩不能只为美观，也要表达状态和优先级。
- 不能只依赖颜色传递信息。

洞察时要看：

- 红、绿、黄、蓝是否承担固定含义。
- 状态是否配合图标、文案或形状。
- 图表颜色是否能区分且不误导。
- 品牌色是否过度使用导致疲劳。

常见问题：

- 所有重点都用品牌色。
- 成功、提醒、进行中颜色混乱。
- 色弱用户无法区分状态。

设计动作：

- 建立语义色：成功、警告、错误、信息、禁用。
- 颜色、图标、文案共同表达状态。
- 限制单屏高饱和色数量。

关联指标： 错误率、误触率、品牌一致性。

## 八、移动端与场景原则

### 23. 单手操作原则

适用场景： 移动端、短视频、社交、支付、即时操作、碎片时间任务。

UI 重点：

- 高频操作应靠近拇指舒适区。
- 关键按钮不能放在难触达区域。

洞察时要看：

- 主 CTA 是否固定在底部或操作顺手位置。
- 顶部按钮是否是低频操作。
- 弹窗按钮是否容易点错。

常见问题：

- 高频操作都在右上角。
- 底部按钮过小或靠近系统手势区。
- 弹窗主次按钮距离太近。

设计动作：

- 主操作放到底部固定区。
- 顶部保留返回、关闭、轻量操作。
- 控制按钮间距和热区。

关联指标： 误触率、完成时长、任务完成率。

### 24. 响应式适配原则

适用场景： 小屏、折叠屏、平板、Web、多端产品。

UI 重点：

- 不同尺寸下信息层级不能崩。
- 关键操作和关键内容必须始终可见或可达。

洞察时要看：

- 小屏下文字是否换行溢出。
- 卡片是否挤压导致信息失真。
- 底部操作是否遮挡内容。
- 横竖屏是否有合理布局。

常见问题：

- 桌面端布局直接缩到移动端。
- 表格在小屏不可读。
- 长按钮文案溢出。

设计动作：

- 设置断点布局。
- 小屏优先保留核心信息和主操作。
- 表格转卡片、横向滚动或摘要模式。

关联指标： 任务完成率、错误率、视觉喜好度。

## 九、体验闭环原则

### 25. 操作闭环原则

适用场景： 创建、生成、保存、发布、支付、授权、提交、审核。

UI 重点：

- 每个关键动作都要有前置理解、操作入口、即时反馈、结果确认和下一步承接。

洞察时要看：

- 用户是否知道操作前会发生什么。
- 操作中是否知道系统正在处理。
- 操作后是否知道结果和下一步。
- 失败后是否知道原因与恢复方式。

常见问题：

- 操作前缺少预期说明。
- 操作后只有 Toast，没有结果承接。
- 失败态只提示失败，不提供解决方式。

设计动作：

- 操作前给出结果预期或确认摘要。
- 操作中提供进度和可取消机制。
- 操作后提供查看、继续、编辑、分享、返回等承接动作。
- 失败态提供重试、修改、联系支持或保留草稿。

关联指标： 任务完成率、路径流失率、二次使用率、错误率。

### 26. Peak-End 体验原则

适用场景： 生成完成页、支付成功页、发布成功页、任务结束页、会员开通成功页。

UI 重点：

- 用户会更记得体验中的高峰和结尾。
- 关键成功时刻要给出清楚、积极、可继续的反馈。

洞察时要看：

- 成功页是否只提示“成功”，还是有结果价值展示。
- 完成后是否有下一步承接。
- 是否让用户感到任务完成且可控。

常见问题：

- 成功反馈太弱。
- 任务结束后没有继续操作。
- 成功页只做装饰，不服务复用和转化。

设计动作：

- 强化成功结果、收益或作品预览。
- 提供下一步主操作和次操作。
- 保留返回、撤销、查看详情等掌控感入口。

关联指标： 任务完成率、二次使用率、视觉喜好度。
