# 输入与工作流

## 目标

用于判断用户提供的材料是否足够，并决定本次视觉洞察应该采用完整输入、半完整输入还是探索输入。

## 输入材料清单

优先收集：

- 需求名称
- PRD
- 需求背景
- 目标角色
- 关键路径
- 交互链路截图
- 当前 UI 或历史版本截图
- 竞品名称、截图或链接
- 品牌规范、设计系统或品牌关键词
- 历史 UI 问题
- 业务指标或评审关注点

## 三种输入模式

### 完整输入模式

用户已提供 PRD、背景、角色、关键路径、链路截图、竞品材料、品牌规范和历史 UI 问题。

直接输出完整视觉洞察报告。

### 半完整输入模式

用户只提供 PRD、背景、关键路径或部分截图。

处理方式：

- 先说明已提供与缺失材料。
- 基于已有信息先做判断。
- 明确哪些判断需要补充材料后才能完成。
- 如果缺少竞品材料，输出竞品采集清单。

### 探索输入模式

用户只提供一句话需求或模糊方向。

处理方式：

- 不直接生成完整结论。
- 先反推可能的业务目标、用户角色和关键路径。
- 输出需要补充的材料清单。
- 输出初步视觉洞察框架。

## 材料不足时的输出格式

```text
已提供材料：
- ...

缺失材料：
- ...

基于现有材料可以判断：
- ...

补充后可以进一步判断：
- ...

建议下一步补充：
- ...
```

## 引导式提问原则

一次最多问 4 个问题，优先问影响判断最大的内容：

1. 这个需求的 PRD 或需求背景是什么？
2. 关键路径是什么？
3. 有没有链路截图、旧版 UI 或交互稿？
4. 是否需要包含竞品 UXUI 分析？

如果用户不想补充，继续基于已有材料输出，并标注判断边界。
