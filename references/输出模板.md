# 输出模板

## 使用方式

当用户需要完整视觉洞察报告时，按本模板输出。材料不足时可以删减模块，但必须说明删减原因和判断边界。

## 一、输入材料判断

```text
已提供：
- ...

未提供：
- ...

本次可判断：
- ...

需补充后判断：
- ...
```

## 二、需求视觉目标

必须把业务目标翻译成视觉目标。

输出：

- 核心业务目标
- 用户最需要被视觉帮助完成的判断
- 视觉设计目标
- 本需求最关键的视觉判断
- 推荐关注指标

## 三、视觉用户故事

拆 2-3 条。不要写“用户”，必须写具体角色。

格式：

```text
作为【具体角色 + 2-3 个强相关属性】，
在【具体场景】下，
我希望【动作层动机】，
以便【深层动机：省时 / 省钱 / 安全感 / 掌控感 / 情绪满足 / 社交认同 / 避免尴尬】。
```

要求：

- 角色按 PRD 反推。
- 场景要具体，避免“日常使用中”。
- 动作层动机和深层动机都要写。
- 禁止“提升效率”“更方便”这类空话。

## 四、关键视觉路径

先用 `→` 串出主路径。

然后用表格输出：

| 路径步骤 | 用户此刻要判断什么 | 视觉要突出什么 | 应弱化什么 | 关键 UI 元素 | 对应指标 |
|---|---|---|---|---|---|

## 五、设计原则命中分析

先列本次命中的原则，再说明为什么命中。

格式：

```text
本次重点命中的原则：
- 首焦原则：用于判断...
- 视觉层级原则：用于判断...
- ...
```

原则必须来自 `视觉原则知识库.md`。如果是业务判断，不要伪装成原则。

## 六、竞品 UXUI 洞察

如果有竞品材料，输出：

- 竞品选择判断
- 竞品关键路径视觉拆解
- 可借鉴视觉策略
- 应规避的同质化或风险
- 给本需求的视觉机会点

如果没有竞品材料，输出：

- 当前未提供竞品材料，不做具体竞品结论
- 建议采集的竞品类型
- 建议采集的页面
- 采集时重点观察的 UI 维度

## 七、当前 UI / 链路视觉洞察

围绕以下维度输出：

- 第一眼落点
- 信息层级
- 视觉动线
- 主 CTA
- 页面分组
- 操作效率
- 状态反馈
- 操作闭环
- 风险与信任
- 品牌一致性
- 历史 UI 问题修复
- 移动端与无障碍

每个维度尽量说明：

- 当前判断
- 证据来源
- 影响
- 建议动作

## 八、视觉洞察 Checklist

用表格输出：

| 维度 | 当前需求需要重点关注什么 | 证据来源 | 优先级 |
|---|---|---|---|

维度包含：

- 信息层级
- 视觉动线
- 操作闭环
- 用户友好
- 操作效率
- 状态反馈
- 风险与信任
- 品牌一致性
- 历史 UI 问题修复
- 移动端适配与无障碍

## 九、可执行设计建议

这是最终最重要的模块。

用表格输出：

| 问题/机会点 | 证据来源 | 路径步骤 | 相关 UI 元素 | UI 设计动作 | 命中原则 | 预期影响 | 验证指标 | 优先级 |
|---|---|---|---|---|---|---|---|---|

要求：

- 每条建议必须能指导 UI 设计动作。
- 每条建议尽量绑定原则和指标。
- 如果属于业务判断，标注“业务判断，不属于原则库直接推导”。
- 优先级使用：高 / 中 / 低。

## 十、分支与边缘路径

列出当前主路径中容易被忽略的特殊状态：

- 新用户 / 老用户
- 有数据 / 无数据
- 成功 / 失败 / 加载
- 权限不足
- 网络异常
- 误触 / 撤销
- 低端机或小屏幕

并说明每个状态在 UI 视觉上要如何承接。

## 十一、下一步需要补充

如果材料不足，最后输出：

- 需要补充的截图
- 需要补充的竞品
- 需要补充的品牌规范
- 需要补充的历史问题或数据
- 补充后可以进一步判断什么
